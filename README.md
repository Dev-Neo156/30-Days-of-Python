| Better for frequently changing data | Better for fixed data        |

## 💻 Key Concepts

### 📌 Dictionaries

* Store data as **key-value pairs**
* Mutable and ordered (Python 3.7+)
* Keys must be unique
* Values can be of any data type

### 📌 Common Dictionary Methods

* `keys()`
* `values()`
* `items()`
* `get()`
* `update()`
* `pop()`
* `popitem()`
* `clear()`
* `copy()`

### 📌 Sets

* Unordered collection
* Mutable
* Stores **unique** elements only
* Does not allow duplicates

### 📌 Common Set Methods

* `add()`
* `remove()`
* `discard()`
* `pop()`
* `clear()`
* `copy()`

### 📌 Set Operations

* Union (`|` or `union()`)
* Intersection (`&` or `intersection()`)
* Difference (`-` or `difference()`)
* Symmetric Difference (`^` or `symmetric_difference()`)

## 🔄 Dictionaries vs Sets

| Dictionaries             | Sets                        |
| ------------------------ | --------------------------- |
| Store key-value pairs    | Store unique values only    |
| Access values using keys | No indexing                 |
| Mutable                  | Mutable                     |
| Ordered (Python 3.7+)    | Unordered                   |
| Keys must be unique      | All elements must be unique |

## 🔄 `for` vs `while`

| `for` Loop                                  | `while` Loop                                  |
| ------------------------------------------- | --------------------------------------------- |
| Used when the number of iterations is known | Used when the number of iterations is unknown |
| Iterates over sequences                     | Executes until a condition becomes false      |
| Simpler for fixed iterations                | More flexible for condition-based repetition  |
